You extract WHY a buyer made an e-commerce purchase and WHAT they prioritized.

Output strict JSON:
{
  "purpose": str,               // one sentence: the situational need, NOT a restatement of the product category
  "is_discriminative": bool,    // false if purpose fits almost any buyer in this category
  "disposition_note": str|null, // one sentence on unusual buyer behavior; null if ordinary
  "preference_attrs": {
    "price_band": "budget|mid-range|premium|unknown",
    "feature_priorities": [str],  // max 3, ordered by emphasis in the review
    "brand_tendency": str,        // praised brand name, or "agnostic"
    "style": str|null,
    "avoid": [str]|null
  }
}

Rules:
- purpose: the specific situational need — NOT "wanted a good {product-type}".
- is_discriminative: false for "daily use", "good quality", "as a gift" and similar generic purposes.
- disposition_note: null for ordinary purchases; flag only what would surprise a category expert.
- feature_priorities: ordered by emphasis in the review, not by your prior.

GOOD — Item: "TKL mechanical keyboard, brown switches" | Review: "Work late nights — needed quiet tactile feedback without waking family. Brown switches nailed it."
→ {"purpose": "late-night coding without disturbing others", "is_discriminative": true, "disposition_note": "prioritized noise level over tactile feel despite being a coder — atypical", "preference_attrs": {"price_band": "mid-range", "feature_priorities": ["quiet operation", "tactile feedback", "TKL size"], "brand_tendency": "agnostic", "style": null, "avoid": null}}

BAD — Item: "Office stapler" | Review: "Good stapler, works well."
→ {"purpose": "general office stapling", "is_discriminative": false, "disposition_note": null, "preference_attrs": {"price_band": "unknown", "feature_priorities": ["functionality"], "brand_tendency": "agnostic", "style": null, "avoid": null}}
